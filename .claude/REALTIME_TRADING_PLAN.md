## 두 모드 비교

| 구분 | 정기 매매 | 긴급 매매 |
|------|----------|----------|
| **주기** | 4시간마다 | 이벤트 발생 시 |
| **트리거** | 시간 (4시간마다 (0, 4, 8, 12, 16, 20시)) | 가격/지표 조건 |
| **분석 깊이** | 종합 (뉴스, 차트, 지표) | 핵심만 (가격, 지표) |
| **AI 프롬프트** | 상세 분석 요청 | 긴급 판단 요청 |
| **목적** | 전략적 포지션 조정 | 급변 대응/기회 포착 |
| **쿨다운** | 없음 (1시간 간격) | 5~15분 |

## 충돌 방지

### 매매 잠금 (Lock)
```python
import threading

trade_lock = threading.Lock()
last_trade_time = None
MIN_TRADE_INTERVAL = 180  # 최소 3분 간격

def can_trade():
    global last_trade_time
    if last_trade_time is None:
        return True
    return (time.time() - last_trade_time) > MIN_TRADE_INTERVAL

def execute_trade(decision, source="scheduled"):
    with trade_lock:
        if not can_trade():
            logger.info(f"Trade skipped: too soon since last trade")
            return False
        
        result = do_trade(decision)
        
        if result:
            last_trade_time = time.time()
            log_trade(decision, source=source)
        
        return result
```

### 우선순위
```
강제 손절/익절 > 긴급 매매 > 정기 매매
```

- 강제 손절(-7%) 발동 시 → 모든 것 무시하고 즉시 실행
- 긴급 매매 실행 후 30분 내 정기 매매 → 스킵
- 정기 매매 실행 후 15분간 → 긴급 트리거 비활성화 (강제 손절 제외)
